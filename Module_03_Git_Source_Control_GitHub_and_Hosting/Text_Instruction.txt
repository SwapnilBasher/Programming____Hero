1) What is Git? What are the advantages of using Git?

Answer: Git is a distributed version control system (VCS) that records changes to files and coordinates work among multiple people. Advantages include:
- Distributed (every clone has full history, works offline)
- Fast operations (commits, diffs, merges)
- Cheap branching & merging for parallel development
- Integrity (commits are cryptographically linked)
- Staging area for fine-grained commits

2) What do you understand by the term 'Version Control System'?

Answer: A Version Control System (VCS) records and manages changes to files over time so you can review, revert, or branch from previous states. Benefits include history, collaboration, safe experimentation, and auditing.

5) Difference between git fetch and git pull

Answer: git fetch downloads remote refs and objects to your local repository but does not change your current branch or working files. git pull performs a fetch followed by a merge (or rebase) into your current branch, updating it immediately. Use git fetch to inspect changes before integrating them, and git pull when you want to update in one step.
