git init

The git init command is used to create a new Git repository. When you run this command in a directory, it initializes a new Git repository by creating a hidden .git folder that contains all the necessary files and metadata for version control.

git add

The git add command is used to stage changes in your working directory for the next commit. You can use this command to add specific files or all changes in the directory to the staging area, preparing them to be committed to the repository.

git commit

The git commit command is used to save the staged changes to the repository. When you run this command, it creates a new commit that includes all the changes that were added to the staging area, along with a commit message that describes the changes made.

git add .

The git add . command is used to add all changes in the working directory to the staging area. This is a convenient way to stage all modified and new files for the next commit without having to specify each file individually.

  • The dot (.) represents all files in the current directory and subdirectories.
  • It stages new files, modified files, and deleted files.
  • It is commonly used when you want to commit all changes in your project at once.
  • Git moves these files to a staging area where they are ready to be committed.
  • Staging = preparing files for commit.
  • Staging is the process of preparing files for a commit.

git commit -m "Initial commit"

The git commit -m "Initial commit" command is used to create a new commit with a message describing the changes being committed. The -m flag allows you to specify the commit message directly in the command line, making it quick and easy to document your changes.

  • The -m flag stands for "message" and is used to provide a commit message.
  • The commit message should be concise and descriptive of the changes made.
  • This command creates a new commit in the Git repository with the staged changes.
  • It is important to write meaningful commit messages for better project history.

git branch -M main

The git branch -M main command is used to rename the current branch to "main". The -M flag stands for "move" and is used to rename branches in Git. This command is often used to set the default branch name to "main" instead of the traditional "master".

  • The -M flag allows you to rename the branch even if the new branch name already exists.
  • This command is useful for standardizing branch names across repositories.
  • Renaming branches can help improve clarity and consistency in version control.
