## ‡ßß) What do you understand by the **Universal Selector**?

The **universal selector** in CSS is written using an asterisk `*`.
It **selects all HTML elements on the page**.

### Syntax

```css
* {
  margin: 0;
  padding: 0;
}
```

### Explanation

* It applies the same style to **every element** (`div`, `p`, `h1`, `img`, etc.)
* Commonly used for **CSS reset** to remove browser default styles

### Example

```html
<p>Hello</p>
<h1>Heading</h1>
```

```css
* {
  color: blue;
}
```

‚û° Both `<p>` and `<h1>` will become **blue**.

---

## ‡ß®) Differentiate between the use of **ID selector** and **Class selector**

| Feature        | ID Selector    | Class Selector    |
| -------------- | -------------- | ----------------- |
| Symbol         | `#`            | `.`               |
| Usage          | Unique element | Multiple elements |
| Can repeat?    | ‚ùå No           | ‚úÖ Yes             |
| Priority       | Higher         | Lower             |
| HTML attribute | `id`           | `class`           |

---

### ID Selector Example

```html
<h1 id="main-heading">Welcome</h1>
```

```css
#main-heading {
  color: red;
}
```

‚û° Used when **only one unique element** needs styling.

---

### Class Selector Example

```html
<p class="text">Paragraph 1</p>
<p class="text">Paragraph 2</p>
```

```css
.text {
  color: green;
}
```

‚û° Used when **multiple elements share the same style**.

üìå **Rule:**

* One page ‚Üí **one ID**
* One class ‚Üí **many elements**

## ‡ß©) How can you use CSS to control **image repetition**?

CSS controls background image repetition using the `background-repeat` property.

### Syntax

```css
background-repeat: value;
```

### Values & Meaning

| Value       | Description                      |
| ----------- | -------------------------------- |
| `repeat`    | Default, repeats both directions |
| `no-repeat` | Image appears only once          |
| `repeat-x`  | Repeats horizontally             |
| `repeat-y`  | Repeats vertically               |

---

### Example

```css
body {
  background-image: url("bg.png");
  background-repeat: no-repeat;
}
```

‚û° The image will appear **only once**.

---

### Common Usage (Best Practice)

```css
body {
  background: url("bg.png") no-repeat center;
}
```

‚ùå **No, they are not the same.**

### HTML Tag

* A tag is just the **markup** inside `< >`
* Example:

### HTML Element

* An element includes:

  * Opening tag
  * Content
  * Closing tag

### Example


‚û° This whole line is an **HTML element**.

---

### Summary

## ‡ß´) Difference between **inline, block, and inline-block elements**

### And: Can an inline element be changed to block?

### Block Elements

* Take **full width**
* Start on a **new line**
* Can set width & height

Examples:

```html
<div>, <p>, <h1>
```

---

### Inline Elements

* Take **only required width**
* Stay on the **same line**
* Cannot set width & height

Examples:

```html
<span>, <a>, <strong>
```

---

### Inline-block Elements

* Stay on the **same line**
* Allow **width & height**

Examples:

```css
display: inline-block;
```

---

### Comparison Table

| Feature        | Block | Inline  | Inline-block |
| -------------- | ----- | ------- | ------------ |
| New line       | Yes   | No      | No           |
| Width/Height   | Yes   | No      | Yes          |
| Margin/Padding | Yes   | Limited | Yes          |

---

### Can we change inline to block?

‚úÖ **YES**

### Example

```html
<span class="box">Hello</span>
```

```css
.box {
  display: block;
  width: 200px;
  background: yellow;
}
```

‚û° `<span>` (inline) becomes a **block-level element**.

---

### Final Tip (Exam-Friendly Line)

> Using the CSS `display` property, any HTML element can be converted into inline, block, or inline-block.

## 1Ô∏è‚É£ Why use `<section>` instead of `<div>`?

### üîπ `<div>`

* **Non-semantic** (no meaning)
* Used **only for styling or layout**

```html
<div class="content">
  <h2>About Us</h2>
  <p>We build websites.</p>
</div>
```

---

### üîπ `<section>`

* **Semantic** (has meaning)
* Represents a **group of related content**
* Improves **SEO & accessibility**

```html
<section>
  <h2>About Us</h2>
  <p>We build websites.</p>
</section>
```

---

### ‚úÖ Why interviewers prefer `<section>`?

‚úî Search engines understand content better
‚úî Screen readers work better
‚úî Cleaner, meaningful HTML

### üìå Simple rule to remember:

> **If content has a heading and meaning ‚Üí use `<section>`**
> **If only for styling/layout ‚Üí use `<div>`**

---

## 2Ô∏è‚É£ Difference between `margin` and `padding`

### üîπ Margin

* Space **outside** the element
* Creates distance **between elements**

```css
.box {
  margin: 20px;
}
```

---

### üîπ Padding

* Space **inside** the element
* Space between **border and content**

```css
.box {
  padding: 20px;
}
```

---

### üîπ Visual Understanding

```
[ Margin ]
  [ Border ]
    [ Padding ]
      Content
```

---

### üîπ Comparison Table

| Margin             | Padding            |
| ------------------ | ------------------ |
| Outside space      | Inside space       |
| Separates elements | Expands element    |
| Transparent        | Background applies |

---

### üî• Interview One-liner:

> **Margin creates space outside an element, padding creates space inside an element.**

---

## 3Ô∏è‚É£ Center a `<div>` Horizontally or Vertically

---

### ‚úÖ A) Center Horizontally (Most Common)

```css
.box {
  width: 200px;
  margin: 0 auto;
}
```

‚úî Works only if **width is set**

---

### ‚úÖ B) Center Horizontally & Vertically (BEST & Modern ‚Äì Flexbox)

```css
.container {
  display: flex;
  justify-content: center; /* horizontal */
  align-items: center;     /* vertical */
  height: 100vh;
}
```

```html
<div class="container">
  <div class="box">Center Me</div>
</div>
```

‚úî Most recommended
‚úî Interview favorite answer

---

### ‚úÖ C) Center Using `position` (Classic Method)

```css
.box {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}
```

‚úî Works everywhere
‚ùå Less clean than Flexbox

---

### üî• Interview Tip:

If asked **‚Äúbest way to center a div?‚Äù**, say:

> **Flexbox using justify-content and align-items**

---

## ‚úÖ Quick Interview Summary

* `<section>` ‚Üí semantic, meaningful content
* `<div>` ‚Üí layout & styling only
* `margin` ‚Üí outside space
* `padding` ‚Üí inside space
* Best centering method ‚Üí **Flexbox**
